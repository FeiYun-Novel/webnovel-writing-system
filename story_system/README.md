# Story System 入口

## 新开对话必读顺序

> **日常续写：只读 `写前速览卡.md`（1 页）+ 按需查本章相关那几段就够**，不必全量通读下面这些（省 token，详见 `检查表/阶段0`）。下面的全量顺序留给"新开大对话写第一章 / 跨度大拿不准 / 怀疑速览卡过期"时。

0. **写前速览卡.md** ← 每章动笔前必读这一张（信息可见性符号 / 各角色专属反应特征 / 少用词 Top / 技法口诀 / 按需查指针）
1. **README.md** ← 本文件
2. **章节进度表.md** — 写到哪了
3. **伏笔清单.md** — 埋了什么、回收了没
4. **人物状态.md** — 角色当前进度/关系（每章更新的活文档）
5. **已发生事件.md** — 关键事件时间线
6. **未解决问题.md** — 还没想好的坑
7. **名场面池.md** — 全书预定要写的标志性桥段/名场面银行（含触发条件，写前查一下别提前写崩）

## 写每章时：按 SOP + 检查表走

- **流程（按什么顺序做）：** `story_system/写作流程.md` —— 阶段 0→5 的 SOP，只写流程。
- **检查表（每步核对有没有犯错）：** `story_system/检查表/` —— 每个阶段一份独立 Checklist，逐条对、不凭记忆。入口见 `检查表/README.md`。
- 两份分开：SOP 只管流程，检查表只管对错。每做完一个阶段，打开对应检查表逐项核对，有问题就改再进下一步。

## 写具体描写时：翻技法库

- **`story_system/写作技法/`** —— 把"某类描写怎么写"沉淀成可套公式。入口见 `写作技法/README.md`。
  - 任何描写（人/物/景/动作） → `白描手法.md`（最底层的描写地基）
  - 内心活动/情绪 → `心理描写五招.md`
  - 日常戏 → `日常场景写法.md`
- 规矩：**写到对应场景前先翻、不凭记忆**；和 `写作风格指南.md`（管腔调/大白话）配合用。

## 角色信息两层结构（重要）

- **角色「设定」（不变的部分）：** 看 `角色/<名字>.md` — 一人一档（本项目示例：主角、感情线角色一~四、反派、系统、导师/长辈NPC 等）。记定位、来历、性格、修为/能力设定、武器、外貌、能力、萌点、弧光、感情线、声音样板等。
- **角色「当前进度/关系」（会变的部分）：** 看 `story_system/人物状态.md` — 每写完一章必须回来更新（当前状态/位置、各角色对主角态度与好感信任度、没说破的心事、最近互动）。
