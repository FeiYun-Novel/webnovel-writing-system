# 阶段 4 检查表 · 写后自检（不过不交稿）

> 目的：成品出厂前最后一道关。**逐条**过 A/B/C，不过不交稿；全过＝**候选定稿**（正式封存＝用户认可＋阶段5完成，见 写作流程.md 总纪律1）。
> 流程见 `../写作流程.md` 阶段 4。

## 本阶段的逐项清单 = `../章节自检清单.md`

阶段 4 的完整检查项（A 结构维度 / B 设定伏笔 / C 历史错题若干条）维护在 **`story_system/章节自检清单.md`**，不在此重复，避免两份不同步。本页只管"怎么用它"。

## 步骤 0 · 文风机检闸门（必须先跑，全绿才继续）

- [ ] 跑 `python3 story_system/scripts/style_gate.py 正文/第X章-xxx.md`，**全绿才继续**；有红项先改写正文再重跑
  - 通用项＝最低字数/破折号/「不是A是B」/明喻密度/单句成段/段内排比/符号配对/文件名H1一致（阈值在脚本 CONFIG 按你自己文风校准，并与你项目风格指南对应小节保持同步）；另有**禁用词/敏感词两个可配置项**（按你的书填词表）
  - 为什么放最前：LLM 靠"读感"数数不可靠（破折号超标数倍、多路审稿代理都可能读不出来）——**凡是数值可判定的规则，一律脚本核验**
  - 红项没清就派审稿代理 = 白花钱（审稿该干的是机器干不了的活）

## 用法（铁律）

- [ ] 把 `章节自检清单.md` 的 A / B / C **逐条拉出来对照本章正文**——一条一条看，**不凭记忆**
  > 教训：凭记忆"粗过"会漏掉清单里早就有的具体条目，被用户点出来才发现。**凭记忆必漏。**
- [ ] **A 结构维度**全过（剧情逻辑 / 人物目标 / 情绪关系 / 身体信息 / 场景转场 / 章末承接——具体每项的判定标准由你自己项目定义）
- [ ] **B 设定与伏笔**全过（人设言行、伏笔埋/收、用上旧钩子、没破坏既有设定）
- [ ] **C 历史错题**（已分类）**按组扫**——每类先看共性领读再过条目，比线性过全部条目召回更高。**具体的历史错题分类与高频重犯类型，以你自己项目的 `章节自检清单.md` C 节为准**（每踩一次新坑就补一条，越写越准）。

## 技法兑现核对（对照阶段2「技法清单」）

- [ ] 阶段2 技法清单里给每个场景/钩子点的招，**正文逐场真的用上了吗**（计划写了招、正文没落实 = 白点）
- [ ] 情绪/钩子/大场景这几处，有没有图省事退回"贴标签 / 铺排比 / 流水账"，把点好的招丢了
- [ ] **阶段2「本章例外与豁免」栏逐场兑现了吗**——计划里批准的每条特殊写法（哪场收吐槽、哪个设定元素本章首露、哪类角色只用外部动作……），正文是不是照计划执行、没多没漏。**计划写对、正文写偏，这条负责拦**（没有这条验收回路，例外栏就只是写给用户看的）

## 不过怎么办

- [ ] 任意项不过 → 按 **deepest-first** 回阶段 3 对应模块修（`plot_logic/character_consistency` > `transition/dialogue/chapter_ending` > `anti_ai_voice`）
- [ ] 修完**重过该项**
- [ ] 全过 → **候选定稿**，交用户过目（关卡②）；用户认可＋阶段5完成后才正式封存 forward-only

## 分工并行自检（多 Agent · 关键章用）

单 Agent 查一整章易"注意力稀释"、漏检细处。**关键章（大高潮/塑造章）封存前，按"出场角色＋维度"拆成多个 sonnet 子 Agent 并行查；主 Agent（Opus）拆活 ＋ 汇总去重 ＋ 定稿。**

这是本系统的核心方法论之一，**不限题材通用**——下表的 5 路是按"读什么/查什么维度"切分，不是按具体人物切分，换成任何小说都能直接套用，只需把角色分组换成你自己项目里对应的角色。

### 拆法（按"读什么"切，5 路，别一角色一 Agent）

| 子 Agent（sonnet） | 查什么（维度，具体标准按你自己项目的角色卡/风格指南/技法库填） | 读什么（决定成本） | effort |
|---|---|---|---|
| **配角/关系线组**（出场的相关角色合一个） | 人设一致性、反差萌尺度、角色专属反应特征的呈现方式、信息可见性格式规范 | 本章＋出场角色的角色卡＋人物状态文档 | medium |
| **主角 +（可选）系统组** | 吐槽/口头禅重复度、外号使用、身份秘密暴露风险、走心场合的称呼选择、系统类角色的说话方式、受威胁时的反应逻辑 | 本章＋主角卡＋系统卡（若有）＋风格指南 | medium |
| **写法/AI味组** | 大白话程度、蛇足点题、文绉绉、标点/句式规范、比喻是否硬凑、技法兑现、章末钩子质量 | 本章＋写作风格指南＋章节自检清单C段 | **high** |
| **剧情/设定组** | 逻辑前提触发后果、伏笔埋收、特殊设定（如先知类）是否失灵、设定一致性、新设定有无铺垫 | 本章＋伏笔清单＋已发生事件＋未解决问题＋当前分卷大纲 | medium |
| **读者视角组** | **纯·第一次读者体验审**——不看角色卡/设定文件，只读本章正文（+上一章最后若干行做衔接），以"第一次看这本书的读者"身份找：①读到哪里一头雾水/逻辑跳跃（"角色凭什么知道这个？""前面没说过啊？"）②作者旁白替角色/读者下了结论（画面已说清还加总结句）③哪句话读起来别扭/指代不清/要回翻才看懂 ④节奏卡壳（某段太长读不下去/某处转折太突然没准备）。**不查设定一致性（那是剧情组的活），只管"读着顺不顺、懂不懂、爽不爽"。** | **仅本章正文＋上一章末尾若干行**（刻意不给设定文件，保持读者盲区） | medium |

- **可选第 6 路·体系一致性组**（打斗/力量体系章加跑）：对照你项目的力量体系设定，查每个出手角色的表现是否符合其等级（威力/消耗/对撞结果/伤势），防"低阶打出高阶效果"
- **跨环境降级（换模型/换工具时读）：** 本编排不依赖特定模型——Claude Code 子代理自动可用；Codex 等工具有子代理但**不会主动拆分**，须在指令里显式要求"把以下 N 路分别派给独立子代理并行执行"；无子代理环境则按本表分 N 次**独立会话**逐路跑（关键是输入隔离——读者视角组绝不能带着设定知识审）
- 每个子 Agent 用 **schema 强制结构化输出**：`findings[{severity 硬伤/建议/吹毛求疵, location, issue, suggestion}] ＋ verdict`。**多组命中同一处 = 高置信**，优先改。
- **务必告诉子 Agent 本章范围**（如"本章剧情只到某个节点、某个关键设定在后面章节才揭晓，别报没到"），免得误报缺漏。

### 5 路 ↔ C 节分类映射

各路子 Agent 直接领对应分类的**具体条目**，不再只拿概括描述（分类编号见 `../章节自检清单.md` C 节，具体映射关系由你自己项目按 C 节的实际分类去定义）：
- **配角/关系线组** → 角色反差与专属反应特征相关分类
- **主角+系统组** → 主角专属信息机制逻辑相关分类 ＋ AI味类中与套路化重复相关的条目
- **写法/AI味组** → AI味类（全部）＋ 留白/钩子/反派类（全部）＋ 硬伤类中的格式规范条目
- **剧情/设定组** → 设定一致性类（全部）＋ 留白/钩子/反派类的伏笔/钩子相关 ＋ 硬伤类的常识硬伤
- **读者视角组** → **不领 C 节条目**（它不对照清单，靠"读着哪里卡"的直觉找问题，与其它4路互补而非重叠）

### 机械项由 style_gate.py 全权负责（人工/Agent 不再数）

凡是能用固定规则/固定词表机械检测的项（禁用词、标点计数、文件名标题一致等），**全部配置进 `scripts/style_gate.py`**，步骤0 跑过即覆盖——别留在人工清单里（人工数不准还浪费，脚本已兜底的项在人工清单里就是纯冗余）。只有"引号内实际字数 vs 叙述所报数字"这类语义层的留给主 Agent 快查。

### 省 token（成本敏感）

- **写法/AI味组喂规则片段**（风格指南要点、C 段），不让它读整本大文件——规则固定、零损失，且它输出最长，省得最多。
- **人设/剧情组保留自读**角色卡＋伏笔（防摘漏、保召回）。
- **读者视角组最省**——只喂本章正文＋上一章末尾若干行，刻意不给设定文件（保持读者盲区），输入最少。
- **分级审稿（制度化）**：审稿路数由阶段2定的**章型**决定——**关键章＝全 5 路**；**过场章＝固定 3 路**（读者视角组不可豁免＋写法/AI味组＋**本章主角色一致性组**——按本章主视角选：主角组/感情线角色组/反派专项/纯配角过场就对照该角色卡，别硬套"二选一"，纯反派章或世界线过场两个常规组都盖不住），**跳过最贵的剧情/设定组**、由**每 3 章一次的一致性巡检**兜底（派 1 路子代理一次性核对最近 3 章正文 vs 伏笔清单/人物状态/已发生事件，结果记入修订记录；若这 3 章全是关键章可跳过该轮巡检）。章型用户有最终裁定权，拿不准报关键章。
- **成本因项目篇幅而异**，实际 token 消耗以你自己项目跑出来的数据为准，可自行记录做参考。

### 汇总 → 改 → 封存

主 Agent 把各组 finding **去重合并成一张按严重度排序的清单**，交用户圈选 → 改（涉及人物内心/感情进度的改动先给用户看句子）→ 重过机械项 → 封存。
