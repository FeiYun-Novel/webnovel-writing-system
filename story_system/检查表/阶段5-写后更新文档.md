# 阶段 5 检查表 · 写后更新文档

> 目的：把本章产生的所有变化同步回状态文档，保证下次冷启动读到的就是最新的书。漏更新 = 下章按旧设定跑偏。
> 流程见 `../写作流程.md` 阶段 5。

## 逐个文档更新（打勾）

- [ ] 章节进度类文档 —— 本章状态标 ✅；填好下一章核心要素；改"最后更新"日期
- [ ] 人物状态类文档 —— 本章出场角色的状态、对主角的态度与好感信任度、没说破的心事、最近一次互动；改"最后更新"
- [ ] 伏笔清单 —— 新埋的加行、推进的更新状态、回收的标 ✅（**回收必须已经用户确认过**）
- [ ] 已发生事件文档 —— 本章关键事件按时间线补进去（只记对后续有影响的）
- [ ] 章节总结文档 —— 写本章总结
- [ ] 未解决问题文档 —— 解决的标 ✅ 并写方案；本章新冒出的坑加进去
- [ ] 修订记录文档 —— 若本章有润色/修订/设定调整，记一行（日期/章节/版本/改了什么/为什么）
- [ ] 若本章调整了**未来章节的规划**（如某伏笔/角色登场改到后面）→ 同步更新分卷大纲和相关伏笔行

## 追踪几件事（对照你项目 CLAUDE.md 的硬约束，逐个确认没漏）

- [ ] 性格 / 关系 有没有变
- [ ] 秘密 还在不在保密（谁知道了）
- [ ] 伤势 是好了还是加重了
- [ ] 物品 有没有易手 / 消耗 / 新增
- [ ] 已知信息 谁又知道了什么（尤其"谁获知了哪条关键信息、产生了什么判断"——若你的项目有类似的信息可见性机制，具体到"谁感知到了哪条内容"）

## 收尾机检（必跑）

- [ ] 跑 `python3 story_system/scripts/consistency_gate.py`，**红项清零才算封存完成**（章号三处一致/伏笔状态自洽/台词引用核对/位置新鲜度/悬空指针/水位/命名——账本漂移当场拦截，不留给下轮审计）；警告级逐条人工过目，裁决为合法转述的进 `scripts/quote_whitelist.txt`
- [ ] **账本记台词的纪律：要么贴正文原句、要么去引号写转述**——改写了还挂引号是最危险的漂移（会让后续写作以为正文交代过不存在的话）

## 收尾自查

- [ ] 状态文档里的描述与正文**最终版**一致，没有残留旧版本的说法（如正文已改、文档还写着旧情节）
- [ ] 重要剧情决定（影响后续的转折/新信息）已落到人物状态文档和已发生事件文档
