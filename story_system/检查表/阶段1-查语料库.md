# 阶段 1 检查表 · 写前查语料库

> 目的：不空手写。先捞几个真实范例看结构，再动笔。
> **这一步是可选增强**：如果你自建/接入了语料检索工具或正反例库，就做；没有的话可以跳过，直接进阶段 2（不影响系统运行）。
> 流程见 `../写作流程.md` 阶段 1。

## 核对项

- [ ] 判断了本章主类型和题材/标签（具体类型划分与标签体系按你自己项目定义）
- [ ] 跑了检索脚本，捞到若干范例（具体检索脚本/语料库路径按你自己项目配置）
- [ ] 总结了这些范例的**共同结构**：怎么起（抓手）、怎么转（节奏/反转）、怎么收（钩子/余韵）
- [ ] 把总结出的结构用到了本章计划里（不是看完就丢）

## 卡壳时（按症状进你自己的正反例库，若已自建）

若你维护了分模块的正反例库（`<你的范例库路径>/<模块>/`），按症状进对应模块查；没有就凭下表症状自己判断该补什么：

- [ ] 对白没张力、像说明书 → `dialogue`
- [ ] 转场生硬、时间跳切、视角切换 → `transition`
- [ ] 章末没拉力、不想追更 → `chapter_ending`
- [ ] 因果断裂、动机不足、触发不成立 → `plot_logic`
- [ ] 人设漂移、关系/情绪断片 → `character_consistency`
- [ ] AI 味、套话氛围、腔调单一 → `anti_ai_voice`

> 原则：**先有结构参照，再动笔。** 专项问题优先调专项模块，别只停在泛检索。
