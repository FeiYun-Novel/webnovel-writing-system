# 阶段 3 检查表 · 写中执行

> 目的：写正文时边写边守节奏、格式、风格，把问题挡在落笔阶段，少留给阶段 4 返工。
> 流程见 `../写作流程.md` 阶段 3。

## 文风锚（写场景前先对样本）

- [ ] **写每类场景前，先读你项目 `文风样本卡.md` 的同类样本再动笔**（样本卡=作者明确认可过的正文选段，按场景类型归档；规则管下限，样本管手感）。**换模型/换工具/新会话时这一步不可省**——抽象规则跨模型迁移损耗大，具体范文几乎无损。模板见 `../模板/文风样本卡-模板.md`

## 节奏

- [ ] 开篇有抓手（冲突 / 悬念 / 笑点 / 危机具象——具体用哪种、开篇多长字数见钩，按你自己的写作风格指南定）
- [ ] 保持内心独白/吐槽的节奏，不要整段整段没有（具体频率按你自己的写作风格指南定）
- [ ] 每场戏角色的生理/动作反应到位（具体是哪类反应、怎么写，由你自己的角色设定与风格指南决定）
- [ ] 视角切换有统一的分隔标记（具体标记形式按你自己项目定）
- [ ] 每场戏都有"变化"，没有就删 / 并 / 压缩
- [ ] **照阶段2「技法清单」逐场套招**（不是临时现翻、更不是凭记忆）：写每场前翻出计划里给这场点的那一招，按对应技法份的方法写。临场发现计划漏点了招 → 当场翻库补上（具体技法份与路由规则见你自己项目的技法库）

## 信息可见性格式（若你的项目有"谁能听到/感知到什么"这类机制）

- [ ] 逐句判断"谁能感知到"，并核对是否用对了你自己项目约定的符号/格式规则（具体符号约定、每种符号对应的可见范围，按你自己项目的写作风格指南定）
- [ ] 不同可见性内容紧邻出现时，用叙事锚点把界限标清楚（谁有反应 / 谁没反应）
- [ ] 感知机制的触发条件是否凑齐（具体条件、谁容易凑齐/谁容易错过，按你自己项目定）；多人在场时注意别让"该错过的人"莫名其妙也感知到了
- [ ] **若你的项目没有类似机制，此节整体跳过。**

## 风格（大白话 + 去 AI 味）

- [ ] 叙述 / 心声 / 内心独白符合你项目定的"大白话"标准，不文绉绉（具体标准、例外场景按你自己的写作风格指南定）
- [ ] **每写完一段当场检查一遍**（别攒到整章！情绪段/钩子段最容易飘，越是这类段落越要守住"大白话"而不是转向文雅词）：对照写作风格指南做具体的词句替换（具体替换清单按你自己项目定）
- [ ] **重点揪易漏的虚词/凑词类别**（grep 抓不到、最容易复发）：具体是哪几类、怎么改，按你自己的写作风格指南定；对白可留一定的身份腔，叙述/心声/内心独白一般不留
- [ ] 用动作 / 对白替代总结，不写空泛的情绪断言
- [ ] 每个人说话带身份感，不一个腔调（具体的角色声音样板按角色卡定）
- [ ] 不写套话氛围、不写工整排比句
- [ ] 角色的专属反应特征通过身体信号"显"而不直接点破具体形态/标签（具体呈现方式、可保留的例外称呼，按你自己项目的写作风格指南定）
- [ ] 反派/负面角色的阴暗面只给动作级暗示，不写计划独白；悬念露影不露人、不加作者点题句
- [ ] **情绪克制检查**（见你自己风格指南的总纲）：走心/沉重/虐段是否收住了不合时宜的调侃、比喻是否为了工整硬凑、含蓄处是否在该停的地方停住、没有画蛇添足补解释句（具体标准按你自己的写作风格指南定）

## 信息克制 + 吐槽不千篇一律

- [ ] **信息别一次说全**：设定/状态/机制/代价别一段"说全说细"（AI 什么都交代完整的毛病），点到读者懂即可（具体阈值按你自己项目定）
- [ ] **吐槽/口头禅等重复元素别用得过密**，同一章内换着角度来（具体频率与花样按你自己项目定）
- [ ] **若你的项目有"先知记忆/预知"类设定**：检查其查阅/引用方式是否符合设定内在逻辑，别写出与设定矛盾的呈现方式（具体呈现方式、直白暴露的雷区，按你自己项目定）
- [ ] **若涉及"预知走到原著没写的地方"类设定**：之后别又表现得像还全部知道，具体空白范围与推进方式按你自己项目定
- [ ] **该藏的机制别提前说破**：核心待揭机制交给角色自己去发现/去猜，别让旁白或其他角色提前点透
- [ ] **多人在场点明说话对象**：人数较多的场景补一句指代，别让读者认错听话人

## 命名规范

- [ ] 文件名与正文标题的命名规则一致（具体命名格式——数字体系、分隔符、间距等——按你自己项目定）
- [ ] 文件名里的标题 = 正文标题，一字不差

> 提示：这份是"边写边对"的快表；写完后仍要走阶段 4 的完整自检，二者不互相替代。
