# 阶段 0 检查表 · 写前读状态

> 目的：开写前把"整本书的最新状态"读进脑子，所有判断以文档为准，不靠对话记忆。
> 流程见 `../写作流程.md` 阶段 0。

## 读法：速览卡优先 + 按需精准读（省 token，别全量通读）

- [ ] **先读速览卡**（1 页，汇总本项目的可见性规则/角色反应约定/风格要点/技法要点等——具体收哪些内容、放在哪个文件，按你自己项目定）。**每章雷打不动只读这一张**，它替代了"把下面这些大文件从头啃一遍"。
- [ ] **再按需精准读**（只读本章相关的那几条，不整篇通读），至少覆盖：
  - 章节进度类文档 → **当前章那一栏**（写到哪、本章核心要素、章末钩子等）
  - 人物状态类文档 → **本章出场角色那几段**（当前状态/对主角态度/好感/没说破的心事/最近互动）
  - 伏笔清单 → **本章要动的那几条**（可埋/推进/回收，回收必须经用户确认）
  - 已发生事件 → **最近 1-2 章**（上一章结尾、时间线到哪）
  - 角色卡 → **本章出场角色**
  - 分卷大纲 → **本章对应那段**
  - 其他"未解决问题/名场面池"类文档 → 相关条（别撞预定安排、别提前写崩）
- [ ] **全量读只在这几种情况**：新开大对话写的第一章 / 跨度很大拿不准 / 怀疑速览卡与大文件不一致。日常续写不全读。
- [ ] 风格细节（具体标准、清单、口诀）→ 用到时才翻写作风格指南对应小节，不每章通读。

## 写前必须确认的关键点

- [ ] **时间线锚点**：本章发生在第几天 / 距关键剧情节点还有多久 / 和上一章结尾怎么衔接（避免时间跳切穿帮）
- [ ] **出场角色当前态**：每个出场角色"此刻对主角的态度、好感、秘密、伤势、持有物"都心里有数（区别于角色卡的静态设定）
- [ ] **信息可见性/感知机制触发条件**（若你的项目有"谁能感知到某类信息"这类机制）：谁在本章满足触发条件（能感知到）、谁虽在场但缺条件（错过）；尤其反派/竞争者等角色是否在场、会不会"撞上"触发（危机伏笔）。**具体触发条件按你自己项目设定填。若你的项目没有类似设定，此项跳过。**
- [ ] **既定设定红线**：本章不碰世界观/体系/角色背景/主线方向（要碰先问用户）
- [ ] 所有上述信息以文档为准，**没在文档里的，不当成已确立的事实**
