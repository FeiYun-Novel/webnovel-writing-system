# 检查表（Checklist）总入口

> **这套文档只写 Checklist（逐项核对有没有犯错）。** 流程（按什么顺序做）见 `../写作流程.md`。
> **SOP 管流程，检查表管对错，两份分开。**

## 怎么用

1. 写一章时，按 SOP 的阶段 0→5 走。
2. **每做完一个阶段，打开对应检查表，逐条核对。**
3. 有不过的项 → 当场改 → 重新核对该项 → 再进下一阶段。
4. **逐条拉出来对，不凭记忆**——凭记忆必漏（已有教训：阶段4自检凭记忆容易漏掉清单里早就写明的具体条目，具体条目以你自己项目的检查表为准）。
5. 费 token，但换来的是返厂率低、成品稳。这是有意的取舍。

## 各阶段检查表

| 阶段 | 检查表 | 一句话 |
|------|--------|--------|
| 0 | [阶段0-写前读状态.md](阶段0-写前读状态.md) | 写前把整本书的状态读进脑子，不靠记忆 |
| 1 | [阶段1-查语料库.md](阶段1-查语料库.md) | 不空手写，先捞范例看结构 |
| 2 | [阶段2-本章计划与确认.md](阶段2-本章计划与确认.md) | 计划讲清、设定变动先讨论、用户确认再写 |
| 3 | [阶段3-写中执行.md](阶段3-写中执行.md) | 边写边守节奏/格式/风格 |
| 4 | [阶段4-写后自检.md](阶段4-写后自检.md) | 逐条过 A/B/C，不过不交稿（指向 `../章节自检清单.md`） |
| 5 | [阶段5-写后更新文档.md](阶段5-写后更新文档.md) | 把本章变化同步回所有状态文档 |

## 维护约定

- 每当踩了新坑、被用户纠正了新错误 → 该错误进 `../章节自检清单.md` 的 C 节（阶段4），同时如果是某个阶段的流程疏漏，就补进该阶段的检查表。
- 检查表的项要"可核对、可判定对错"，不写空泛原则（空泛原则放 SOP 总纪律）。
- **这套检查表只定"每阶段查哪些维度"（框架），不定"具体怎么写/怎么改"（手艺）。** 具体的口诀、例句、频率数字、替换规则，都放在你自己项目的写作风格指南 / 技法库 / 语料库里，检查表里只留指向它们的指针。
