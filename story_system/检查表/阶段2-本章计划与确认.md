# 阶段 2 检查表 · 本章计划与确认

> 目的：动笔前把本章想清楚、把设定变动和用户对齐，避免写完才发现方向错、返厂重写。
> 流程见 `../写作流程.md` 阶段 2。

## 计划要素（每项都要有）

- [ ] **章型＋审稿风险标签＋本章触碰清单**：记录本章实际涉及的角色关系、信息权限、设定伏笔、连续状态、复杂因果和战斗等级；阶段4据此追加审稿路线，不按章型固定凑路数。随计划报用户确认，**用户有最终裁定权**

- [ ] **本章目标**：一句话说清推进什么（主角这章想做什么）
- [ ] **本章冲突**：主角想做什么 / 谁或什么阻碍他
- [ ] **场景列表**：每场都标 目标—阻碍—**变化**；没有"变化"的场景 → 删 / 并 / 压缩
- [ ] **伏笔操作**：逐条标 🌱新埋 / 🔄推进 / ✅回收
- [ ] **章末钩子**：读者看完最想追问什么；优先信息/悬念/冲突钩，不要只收在情绪上
- [ ] **节奏配比**（若你的项目有某类固定节奏元素，例如喜剧节奏/爽点/信息外露）：哪场戏承载这类节奏、确保全章有相应节奏安排。**具体节奏要素与配比由你自己项目定义。**
- [ ] **视角安排**：每场是谁的视角，切换点在哪
- [ ] **爽点门槛四答**（男频/爽文向建议必答；答不出第 1 问＝计划打回）：①本章主角的"赢"是什么（智斗/嘴炮/算计/信息差/截胡都算，不必是武力）②本章爽点档位（对照你自己分卷的配平表，避免连续多章平路）③憋/放状态（这几章攒了什么憋屈、这章放不放、账欠了几章）④消音自查——赢了却让读者爽不到的常见写法：执行权全外包且无人见证／金手指在成就瞬间自贬泄压／升级零见证（当场没人看见且后文不补"延迟见证"）／全章最大爽点与主角无因果／**认知功劳被工具代劳**（金手指/系统/军师类角色把关键推理结论直接讲给主角听，主角只负责提问和惊讶——读者感受到的是"工具很聪明"而不是"主角很聪明"；正确做法是工具只给原始信息/数据，推理结论由主角自己拼出来）。具体尺度由你自己项目校准
- [ ] **本章例外与豁免栏**（随计划交用户）：一栏列清本章获批的特殊写法——哪几场收着写（走心/严肃场豁免喜剧节奏）、本章首次露出的新设定元素是什么、哪类角色只用外部动作、其他豁免——**写中只认这张计划里的例外，不用回头翻全库对例外**；阶段4 有对应验收条核对逐场兑现
- [ ] **本章技法清单**（＝"写前参考技法库"的强制落点 · 雷打不动每章做，不是临时才查）：四小步逐项打勾——
  - [ ] ① **预判**本章拆成哪几个场景（含章末钩子）
  - [ ] ② 打开 `写作技法/技法分流器.md`：先答**每章固定二问**（谁被蒙在鼓里／读者完成·推进了什么），再给每场**逐问过判定问题清单**——命中即入清单、路由精确到**节**，不许凭记忆挑卡、不许跳问（产出物只记命中项）
  - [ ] ③ **只翻命中的那几节、照公式给这场点名具体招式**——不通读整卡（省 token）；**点不出招 = 还没真去翻**
  - [ ] ④ 按分流器的产出物模板把清单**写进计划**，随计划交用户

## 硬约束自查（碰到就先停下问/讨论用户）

- [ ] **改核心设定**（世界观/体系/角色背景/主线方向）→ 先问用户，不擅自改
- [ ] **回收已有伏笔** → 先经用户确认，不单方面决定
- [ ] **新角色登场 / 角色重大行动的逻辑**（动机够不够、时机对不对、理由读者信不信）→ 先和用户把合理性讨论透
- [ ] **发现前后矛盾** → 先指出问题、列出矛盾点让用户决定，不强行改写

## 出闸条件

- [ ] 计划已用清晰的分点形式给到用户（目标+冲突+场景+伏笔+章末钩子+**技法清单**）
- [ ] 计划里含「本章技法清单」（每个场景/钩子都点了招）——用户据此就能监督"参考技法库"到位没
- [ ] **已获得用户确认**，再进阶段 3 写正文（不先斩后奏）
