# 写作流程（SOP · 每章固定执行）

> **这份文档只写 SOP（按什么顺序做什么）。**
> 每一步"具体要核对什么、有没有犯错"在对应的**检查表**里：`story_system/检查表/`。
> **SOP 管流程，检查表管对错，两份分开，互不夹带。**
>
> 分工：本文件＝详细流程；`CLAUDE.md`＝精简版＋硬约束＋指针。流程以本文件为准，硬约束以 CLAUDE.md 为准。
> 本文件与配套检查表是通用写作工程系统，方法论不限题材——换成你自己的小说设定填充即可。

---

## 总纪律（贯穿全程的三条原则）

1. **停止条件（Loop 停止模块 · 双向封顶）：**
   - **做完即停（forward-only · 防返厂）：** 过了阶段 4 自检的 A/B/C 三节 = **本章封存，不再回头改**。"自检通过"是客观停笔信号，用它取代"我还不满意"的主观感觉。
   - **改不动也要停（2 轮封顶 · 防空转）：** 同一处问题**最多修 2 轮**；2 轮仍不过自检 → **停手**，列出卡在哪、试过哪两招、给两个备选方向，交用户定，**绝不自己硬猜硬改**（防越雕越死、越改越文艺、空烧额度）。
2. **deepest-first 修稿：** 一个问题跨多个模块时，先修最底层，再修表层——
   `plot_logic / character_consistency`　>　`transition / dialogue / chapter_ending`　>　`anti_ai_voice`。
   **别把结构问题当文风问题改。**
3. **只服务本章目标：** 不临时发明会破坏已记录设定的大转折；要改设定 / 回收伏笔 / 新角色登场逻辑，先和用户把逻辑讨论透再写（见 CLAUDE.md 硬约束）。

## 阶段总览

| 阶段 | 做什么 | 产出 | 对应检查表 |
|------|--------|------|------------|
| 0 | 写前读状态 | 脑子里有整本书的最新状态 | `检查表/阶段0-写前读状态.md` |
| 1 | 写前查语料库 | 2-4 个范例的共同结构 | `检查表/阶段1-查语料库.md` |
| 2 | 写本章计划并确认 | 经用户确认的本章计划 | `检查表/阶段2-本章计划与确认.md` |
| 3 | 写正文 | 本章初稿 | `检查表/阶段3-写中执行.md` |
| 4 | 写完过自检 | 自检全过的定稿 | `检查表/阶段4-写后自检.md`（= `章节自检清单.md`） |
| 5 | 写完更新状态 | 所有状态文档与正文同步 | `检查表/阶段5-写后更新文档.md` |

> **用法：** 每做完一个阶段，打开对应检查表逐项核对，有问题就改，改完再进下一阶段。**检查表要逐条拉出来对，不凭记忆**（凭记忆必漏，已有教训）。

## 阶段 0 · 写前读状态（必做，速览卡优先 + 按需读）

**先读 `story_system/写前速览卡.md`（1 页）**，再**按需精准读**本章相关的那几条（进度表当前章栏 / 人物状态本章出场角色段 / 伏笔清单要动的几条 / 已发生事件最近 1-2 章 / 出场角色卡 / 当前分卷大纲对应段）——**不每章全量通读所有大文件**（省 token）。同时锚定本章时间线（第几天 / 距关键事件还有多久 / 和上一章怎么接）。全量读只在"新开大对话第一章 / 跨度大拿不准"时做。**所有判断以文档为准，不靠对话上下文。**

→ 逐项见 `检查表/阶段0-写前读状态.md`。

---

## 阶段 1 · 写前查语料库（skill 硬规则，必做）

判断本章类型与标签 → 跑检索脚本捞 2-4 个真实范例 → 总结它们的共同结构（怎么起、怎么转、怎么收）→ 再动笔。卡在某个专项就进对应模块的正反例库。

```bash
# 如有你自己的语料/范例检索工具，按你自己的安装路径运行，例如：
cd <你的语料检索工具路径>
python3 <检索脚本> --type '<类型>' --tag '<标签>' --limit 4
```

→ 逐项见 `检查表/阶段1-查语料库.md`。

---

## 阶段 2 · 写本章计划（经用户确认）

列出本章计划：目标 / 冲突 / 场景列表（每场 目标—阻碍—**变化**）/ 伏笔操作 / 章末钩子 / 心声配比 / **本章技法清单**。**涉及改设定、动既有伏笔、新角色登场逻辑的，先和用户把合理性·时机·动机讨论透。**

> **新设定/新角色/新势力/新道具登场前，必查前文铺垫（通用铁律）：** 任何第一次出现的新东西（某势力、某人、某物、某概念），前文（对话 / 描写 / 旁人提一两嘴）若完全没埋过 → **先提前一章埋、或本章内先铺一两嘴再用，严禁零铺垫突兀冒出**（"突然冒出来"是 AI 味重灾区，读者出戏）。规划本章时就把"要引入什么新东西、前面铺过没、没铺怎么补"排查掉。

> **「本章技法清单」＝强制参考写作技法库的产出物（这是"写前参考"真正发生的地方）：** 对照 `写作技法/README.md` 的"写什么翻哪份"表，给**每个场景 + 章末钩子点名要用的具体招式**（白描·标志性细节 / 镜头·全景串场 / 心理·逻辑链条 / 日常·钩子……）。填这栏时必然要翻开对应那份技法文档看公式——靠"产出一份清单"逼出参考，而不是靠"记得要参考"。它和计划一起交用户确认，用户能直接看出参考到位没。

→ **这里就是「关卡①」（见上"分步确认关卡"）：计划 + 建议思考深度 + 上下文水位一起报给用户，停下等用户切档 + 点头，才进阶段 3。绝不抢跑。** 逐项见 `检查表/阶段2-本章计划与确认.md`。

---

## 阶段 3 · 写正文（写中约束 + 卡壳调模块）

边写边守节奏约束（抓手 / 心声节奏 / 角色专属反应特征 / 视角切换 / 每场有变化）、信息可见性格式、大白话与去 AI 味。卡壳就调对应模块的正反例库。

### 卡壳路由（症状 → 调你自己的范例/正反例库，按需 `Read`）

如果你维护了一套分模块的正反例库（`<你的范例库路径>/<模块>/`），卡壳时进对应模块先读它的说明，再挑几个正例 + 一两个反例参照。下表只给"什么症状调哪类模块"的路由骨架，具体例库由你自建。

| 症状 | 调哪个模块 |
|------|------|
| 因果断 / 动机不足 / 后果不落地 | `plot_logic` |
| 人设崩 / 目标漂移 / 情绪关系断片 / 身体声音不连续 | `character_consistency` |
| 时间空间跳切 / 情绪承接 / 视角切换 / 章末接下章 | `transition` |
| 对白没刀 / 人物一个腔调 / 信息硬塞 | `dialogue` |
| 章末拉力不够 / 钩子弱 | `chapter_ending` |
| 空泛总结 / 套话 / 形容词堆砌 / 太工整 | `anti_ai_voice` |
| 开头没抓手 / 卖点露头慢 | `opening` |
| 章纲散 / 卷末没兑现 | `volume_outline` |

> 深修顺序（deepest-first）：`plot_logic`/`character_consistency` > `transition`/`dialogue`/`chapter_ending` > `anti_ai_voice`。
> 写**描写本身**（白描/镜头/心理/日常）走项目自带 `写作技法/`；写**结构/对白/钩子**卡壳才进 skill 模块。两者分工：技法库管"怎么写好这一句"，skill 模块管"这一段结构对不对"。

→ 逐项见 `检查表/阶段3-写中执行.md`。

---

## 阶段 4 · 写完过自检（必做，不过不交稿）

把 `story_system/章节自检清单.md` 的 **A（结构六项）/ B（设定伏笔）/ C（历史错题）逐条拉出来对**。任意不过 → 按 deepest-first 回阶段 3 对应模块修 → 重过。全过 → **本章封存（forward-only）**。

→ 逐项见 `检查表/阶段4-写后自检.md`（指向 `章节自检清单.md`）。

> **关键章（大高潮/塑造章）增强 · 分工并行自检：** 把本章按"出场角色＋维度"拆成多个 sonnet 子 Agent 并行查、主 Agent（Opus）拆活＋汇总去重＋定稿，比单 Agent 查一整章召回率高、漏检少（反差萌/蛇足/上帝视角这类细处尤其）。机械项（特征词/破折号计数/字数）主 Agent 自己 grep、不派 Agent。**拆法 / 省 token / 成本** 见 `检查表/阶段4-写后自检.md`。日常过场章可只用 2 路或主 Agent 单查。

---

## 阶段 5 · 写完更新状态（必做）

> **⚠️ 这一步前是「关卡②」（见上"分步确认关卡"）：** 阶段 3 正文 + 阶段 4 自检结果先给用户过目、得到认可，才动状态文档。没认可不更新——免得用户还要改正文、状态却已写死、白同步一遍。

把本章产生的所有变化同步回状态文档：章节进度表 / 人物状态 / 伏笔清单 / 已发生事件 / 章节总结 / 未解决问题 / 修订记录。**确认状态文档与正文最终版一致，无残留旧描述。**

→ 逐项见 `检查表/阶段5-写后更新文档.md`。
