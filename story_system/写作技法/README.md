# 写作技法库（写对应场景前必翻）

> **占位说明：** 本技法库下各份文件均为**占位指引版**，不含作者本人整理的具体技法正文——每份只告诉你"这份该放什么方向的内容"，具体招式/口诀/例句需要你自己填充积累。下面的路由表本身是有用的骨架，予以保留：遇到什么场景该翻哪一份，照着查就行；翻开之后看到的是"该往哪个方向补"的框架，不是现成答案。
> 具体怎么从零开始建立自己的技法库，见 [如何建立你自己的技法库.md](如何建立你自己的技法库.md)。
>
> 🚀 **新手别被 16 份空文件吓到——不用一次填满。** 先填你题材最高频的 **3-5 份**就能开写，建议起步顺序：① 白描手法（所有描写的地基）② 心理描写五招 ③ 对话写作四招 ④ 日常场景写法 ⑤ 开头10大毒点。其余留空、边写边补。

> **用途：** 把"具体某类描写怎么写"沉淀成可套的公式。写到对应场景时，先翻这里看一眼方法，再动笔。
> 这不是设定、不是进度——是**手艺**。和你项目自己的写作风格指南（管腔调/大白话）配合用：风格指南管"用什么词"，本库管"怎么搭结构"。

## ⭐ 每章入口：先跑分流器

**阶段2 定计划时不再"对着下表凭感觉挑"**——打开 [技法分流器.md](技法分流器.md)：每章固定二问 + 逐场过判定问题清单，命中即入清单、路由精确到节。下方路由表是**人工查阅/维护用总目录**（新卡入库时和分流器两边同步）。

> 为什么要分流器：技法库超过十份后，"凭感觉挑"会退化成"永远只翻最熟的两份"，冷门卡变死卡。机械判定保证每份卡在它该出场的章节被想起来。

## 硬规矩

1. **写前先翻、不要凭记忆。** 这些技法的全部价值在于"每次都对一遍公式"，记忆里的版本会缩水。路由用分流器，不靠联想。
2. **方法论服从你自己项目的机制。** 每份文件里的骨架只给方向，不给具体招式——落地时把骨架里的每一条换成你自己项目的题材、机制、人设去填充，不必套用别人的具体设定。
3. **和你项目自己的检查表/写作流程联动。** 写中卡壳、写完自检，都可以回来翻这里对公式。
4. **防复胖。** 单份卡过胖（如超过 ~8k 字符）先拆再追加；「本项目怎么用」一类落地节只写"招式→本项目工具"的映射，**禁写人物关系/感情线当前进度**——进度的唯一权威源是你的人物状态文件，写进技法卡的进度必过期变脏数据。
5. **入库三同步。** 新技法入库时：①归入合适的类（追加进同类卡或新建）②本目录路由表加一行 ③若属新场景类型，给 `技法分流器.md` 加一问。文件一律平放本目录（别建子文件夹，防断链），分类只体现在目录表里。
