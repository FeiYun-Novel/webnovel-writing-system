# 技法分流器（每章阶段2的机械路由入口）

> **解决什么问题：** 技法库建到十几份之后，"凭感觉挑卡"会退化成"永远只翻最熟的那两份"——AI 和人都一样。分流器把"这章该用哪些技法"变成**机械判定**：逐问过一遍固定问题清单，命中哪问就路由到哪份技法卡的哪一节，不命中就不翻。**判定是机械的，写作才是自由的。**
>
> **占位说明：** 下面的"固定二问"和判定问题清单结构是可直接搬用的骨架；具体每一问的问法、路由到哪份卡哪一节，要按**你自己技法库的实际卡片**来建。原则：**你技法库里每一类场景卡，都应该在这里有一问对应**——有卡没问＝那份卡永远不会被想起来。

## 用法（阶段2 定计划时跑，产出物＝本章技法清单）

### 第一步 · 每章固定二问（不分场景，每章必答）

1. **"这一章，谁被蒙在鼓里？"** ——信息差是期待感的地基。答出"谁不知道什么"，就知道该翻信息差/期待感类的卡；答不出，说明这章可能没有钩子，回炉。
2. **"读者在这一章完成 / 推进 / 确认了什么？"** ——只有"什么都没变的纯铺垫"才算答不出；答不出＝这章的存在价值存疑，先回炉再谈技法。

### 第二步 · 逐场过判定问题清单（一场一遍，命中即入清单）

把本章拆成场景（含章末钩子），每场把下面的问题**从头到尾全扫一遍**——只记命中项，但不许跳问（跳问＝凭感觉，分流器就白建了）。

> 下表是**示例结构**，问题数量与内容按你自己的技法库定制。每问三列：判定问题 → 命中路由到哪份卡（精确到节更好）→ 备注（例外/红线）。

### 第三步 · 恒定底盘（不用问，每章都在生效）

白描、节奏留白、去 AI 味这类"句子手艺底盘"不进判定表——它们不是"某些场景才用"，是每一段都在生效。写中卡壳时回底盘卡，不用等分流器点名。

## 产出物模板（写进本章计划，随计划交用户）

```
本章技法清单（分流器产出）：
- 固定二问：谁被蒙在鼓里＝……；读者推进了＝……
- 场景1（xxx）：命中 Q_/Q_ → 翻《xx卡》§x，本场用招：……
- 场景2（xxx）：命中 Q_ → 翻《xx卡》§x，本场用招：……
- 章末钩子：命中 Q_ → ……
（未命中的问不列——清单只记命中项，但确认已全扫）
```

## 硬规矩

1. **逐问全扫、只记命中。** 不许"看一眼场景直接想到卡"——那就是分流器要杀掉的凭感觉路由。
2. **路由精确到节。** "翻《战斗卡》"不合格，"翻《战斗卡》§微观五招"才合格——省 token 且真的会去翻。
3. **点不出招＝还没真去翻。** 清单里每场必须落到具体招式名，落不到就是走过场。
4. **入库同步。** 技法库每新增一类场景卡，本表必须同步加一问（见技法库 README 的入库同步规矩）——否则新卡是死卡。
