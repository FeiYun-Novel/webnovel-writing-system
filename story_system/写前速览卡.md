# 写前速览卡（纯框架模板 · 每章动笔前只读这一张）

> **用途：** 把每章雷打不动要记住的规则压成一页，**替代**"全量通读那几万字状态文件"。
> 看完这页 + 按需查本章相关的那几条（见 §5），就能动笔。**要细节才回大文件，平时只看这一页。**
> 动态状态（写到哪 / 本章钩子 / 谁出场）→ 查 `章节进度表.md` 当前章那一栏。
>
> **说明：** 本文件只保留分区结构（信息可见性符号 / 角色反应特征 / 少用词表 / 技法口诀 / 按需查指针），具体规则内容已全部清空为占位，请替换成你自己项目的规则。

---

## 1. 信息可见性符号（结构占位，逐句判断"谁能听到"）

> 如果你的项目有类似"谁能听到/看到某条信息"的机制，用一张符号对照表压缩规则。符号与判定条件请自建：

- （自定符号1）= 谁能感知：（自定）
- （自定符号2）= 谁能感知：（自定）
- （自定符号3）= 谁能感知：（自定）
- （自定符号4）= 谁能感知：（自定）
- **铁律：** （占位，如果你的机制里存在"当事人不知道信息在外泄"这类设定，把具体判定条件写在这里；本系统不提供内容）

## 2. 各感情线角色专属反应特征（每场至少一个角色有生理/行为反应）

> 结构：每个角色对应一种独特的外形/动作特征，正文不直白点破设定层面的隐喻，靠动作/神态去显。

- 角色一 = （自定特征类型）
- 角色二 = （自定特征类型）
- 角色三 = （自定特征类型）
- 角色四 = （自定特征类型，数量可自建）
- 外号（如有，仅限特定场合使用）：（自定，每人一个，别脸谱化）

## 2.5 反派（伪装型）前期处理提醒（写到他必看）

> （占位）如果你的反派前期需要维持伪装人设，这里应该写你自己关于"内心独白尺度、失态怎么写、伪装怎么秒收"的具体做法。本系统只提示"需要有这类提醒"，不提供具体写法。

## 3. 大白话比例 + 少用词高频提醒（占位）

> （占位）此处填写你自己的：大白话/书面语比例标准、少用词高频清单示例、情绪重场的用词铁律。本系统不提供具体词条与规则内容。
> 全表见 `写作风格指南.md §六`

## 4. 技法库口诀（占位 · 建完技法库后回填）

## 5. 写前按需查（别全读，只读本章相关那几条）

| 要什么 | 去哪 | 只读 |
|---|---|---|
| 写到哪 + 本章钩子 + 核心要素 | `章节进度表.md` | 当前章那一栏 |
| 本章要动的伏笔（埋/推进/回收） | `伏笔清单.md` | 要动的那几条（回收先问用户） |
| 出场角色当前态/好感/秘密/伤势 | `人物状态.md` | 本章出场的那几个 |
| 最近发生了什么、时间线到哪 | `已发生事件.md` | 最近 1-2 章 |
| 出场角色的声音/萌点/外貌/武器 | `角色/<名字>.md` | 本章出场角色 |
| 本章在卷内结构的位置 | `分卷大纲/` 当前卷 | 本章对应那段 |
| 没想好的坑 / 别撞的名场面 | `未解决问题.md`、`名场面池.md` | 相关条 |
| 硬约束 | `CLAUDE.md` | 改设定/回收伏笔先问用户 |
| 核心机制怎么运作（如信息可见性/特殊契约等） | `设定/世界观/` 对应文件 | 拿不准细节时 |

> 拿不准、跨度大、或新开对话第一章时，再回大文件全量读；日常续写只走这张卡 + 按需查。
