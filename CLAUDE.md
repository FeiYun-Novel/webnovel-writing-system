# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

> **本文件为模板。** 保留骨架和方法，具体人名/题材为占位符或示例，请替换成你自己项目的设定。

## 项目性质

这不是代码仓库。这是一个**网文写作项目**（示例题材：题材A + 核心机制B + 信息差机制C——可替换为你自己的题材组合，具体填法见 `EXAMPLE.md`）。所有文件都是 Markdown。

## 冷启动（最重要）

新对话的第一句话必须是：

```
读 story_system/README.md，然后从当前章节继续。
```

这会按顺序加载：章节进度 → 伏笔清单 → 人物状态 → 已发生事件 → 写作风格指南 → 当期章节大纲。

永远不要依赖对话记忆中的上下文——所有状态都存在 `story_system/` 的 Markdown 文件中。

## 写作格式规范（示例：信息可见性约定）

> 如果你的项目有类似"谁能听到/看到什么信息"的机制，参考下表设计自己的符号约定；没有则删除本节。

| 类型 | 格式 | 谁能听到 |
|------|------|----------|
| 主角自言自语/独白 | `（text）` | 部分角色（示例：信息差机制生效时能听到） |
| 主角对专属机制/工具说话 | `（text）` | 无人 |
| 该机制的提示音/反馈 | `【text】` | 仅主角 |
| 口头发言 | `「text」` | 在场所有人 |

核心规则（示例）：主角与专属机制对话时 = 频道加密，其他人什么都听不到。只有主角独白/吐槽时才广播。

## 写作流程（每章）

> **完整 SOP 见 `story_system/写作流程.md`；每步检查表见 `story_system/检查表/`（每个阶段一份，逐条核对、不凭记忆）。每章照它走。** 以下是必守要点。

**总纪律：** 完整初稿和机检通过后，由不同于主笔提供方的模型按 `story_system/检查表/跨模型写后审稿SOP.md` 审稿；全部适用检查通过 = **候选定稿**，交用户过目。**用户认可＋状态文件更新（阶段5）完成后才正式封存，forward-only 不回头改**。修稿先底层后表层：先剧情逻辑/人物一致性，后转场/对白/章末衔接，最后去 AI 味。

**写前（强制）：**
1. **读状态**：`章节进度表.md`、`人物状态.md`、`伏笔清单.md`、`已发生事件.md`、`未解决问题.md`、本章出场角色的 `角色/<名字>.md`、当前 `story_system/分卷大纲/`、`写作风格指南.md`。
2. **查语料库**（如有对应工具/skill）：捞几个真实范例看结构再写。
3. **查写作技法**（建议每章都做）：预判本章拆成哪几个场景 → 对照 `story_system/写作技法/README.md` 路由表给每场翻对应技法 → 照公式给每场点名具体招式，产出「本章技法清单」。**写前学手法、不是写完回头补**，专治 AI 味。
4. **先给本章计划**（目标+冲突+场景列表+伏笔操作+章末钩子+**技法清单**），经用户确认后再写正文。

**写中：** 前 300 字给抓手；按固定频率给一次心声/内心吐槽；每场戏至少一个感情线角色有专属反应特征触发；视角切换用 `---`。卡壳调对应模块正反例库。

**写后（强制）：** 先过机检，再按 `story_system/检查表/阶段4-写后自检.md` 和 `story_system/检查表/跨模型写后审稿SOP.md` 执行跨提供方审稿；**不过不交稿**。用户认可后再更新 `人物状态.md`/`伏笔清单.md`/`章节进度表.md`/`已发生事件.md`/`章节总结/`/`未解决问题.md`。

## 硬约束（不可违反）

1. **不要擅自改变核心设定。** 世界观、修炼体系/能力体系、角色背景、故事主线方向——改之前必须先问用户。
2. **不要擅自揭晓伏笔。** 伏笔的回收必须经过用户确认。写新内容时可以建议埋新伏笔，但不能单方面决定回收已有伏笔。
3. **如果发现前后矛盾，先指出问题，不要直接强行改写。** 列出矛盾点，让用户决定怎么修。
4. **重要剧情变化必须记录下来。** 任何影响后续剧情的决定、转折、新信息，写完立刻更新到 `人物状态.md` 和 `已发生事件.md`。
5. **人物性格、关系、秘密、伤势、物品状态都要持续追踪。** 每章写完后更新 `人物状态.md`，不只看位置和情绪，还要确认秘密是否还在保密、伤势是否痊愈、物品是否易手。

## 风格约束（示例，请替换为自己的比例/规则）

- 日常轻松搞笑（示例 70%），关键时刻走心/燃/虐（示例 30%）
- 禁止空泛总结、套话氛围、说明书对白、工整排比句
- 用动作替代总结，用对白推进剧情
- 每个感情线角色有独特的专属反应特征（设定层面可以有具体隐喻，如动物/元素等）——**但正文尽量不直接点破这个隐喻本身**，靠形状质感＋动作神态暗示
- 主角可以用反差视角吐槽（如现代职场黑话），给每人起外号（自定，别脸谱化）
